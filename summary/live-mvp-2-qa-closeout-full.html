7QA Recommendationข้อเสนอแนะจาก QA — แยกตามบริบทการปล่อย (ปล่อยให้ใคร · ที่ไหน)

คำตัดสิน “พร้อมปล่อย” ไม่ใช่ค่าเดียว — ขึ้นกับ audience + environment · build เดียวกันได้ 2 คำตอบ (เกณฑ์คนละชุด · ไม่ได้ขัดกันเอง):

🌐 Prod สาธารณะ (คนทั่วไปใช้จริง) — 🔴 Not Ready
  • ⚠️ Executed 0% green · script tier = 0 · story ส่วนใหญ่ยัง To Do
  • 🔴 External blocker ×3: Engine RTMP egress (P0) · Engine tool-calling (P1) · Hub metering ingest (P1)
  • 🔴 Epic 8 (AIEventLog foundation) ยังไม่ deliver → ขวาง Epic 10/11/15
  • ⚠️ MVP-1 regression ยังไม่ได้รัน — ยืนยันของเดิมไม่พังไม่ได้
🧪 UAT / pilot-internal (design partner) — 🔴 Not Ready (ยังไม่ถึงขั้น pilot)
  • เกณฑ์ pilot ต่ำกว่า prod = แค่ core loop เวิร์ก + automation/contract เขียว (design ~100%)
  • MVP-2 มี design ~100% ✅ เท่า MVP-1 — แต่ automation = 0 script · contract เขียนไม่ได้ (ไม่มี frozen artifact) · core loop ใหม่ยัง To Do / 0 green
  • → ยังไม่มี feature MVP-2 ที่รันเขียวให้ partner ลอง (ต่างจาก MVP-1 ที่ core Live loop เขียวแล้ว) → ยังไม่ถึงขั้นส่ง pilot

เงื่อนไขพลิกสถานะ — ทำครบ → 🟡 UAT/pilot ก่อน แล้วจึง 🟢 prod สาธารณะ:

  1. เคลียร์ external frozen artifact: Engine tool-calling (D7) + RTMP egress (D8) + Hub ingest (D6) → เขียน contract test red-first
  2. dev ปิด story P0 ตาม build order (Epic 8 → 9/10/11) + เขียน integration script
  3. QA เขียน + รัน e2e script บน stack จริง (แก้ stack + 2FA) ให้ผ่านเขียว → core loop MVP-2 เขียว = ถึงเกณฑ์ 🟡 pilot
  4. รัน MVP-1 Regression suite (7 Epic) ให้ผ่าน — ยืนยันไม่ regress
  5. ผ่าน mandatory negative test: impersonation no-cross-tenant-leak (NFR-005) + PDPA consent gate (Epic 10) → ถึงเกณฑ์ 🟢 prod
